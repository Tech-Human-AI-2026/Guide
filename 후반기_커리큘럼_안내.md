# 인간중심의 AI — 후반기 커리큘럼 V4

전반기(1~7주)에 우리는 거버넌스를 배웠다. 인권 리스크, 문서화 프레임워크, 참여형 데이터 스튜어드십.

후반기는 그 "어떻게"에 답한다. 추상적 원칙 대신 **실제 오픈소스 도구**를 손에 쥐고, AI가 인간의 통제 안에서 일하도록 만드는 엔지니어링을 배운다.

## 인간중심의AI

| 전반기에 배운 원칙 | 후반기에 구현하는 기술 |
|-------------------|---------------------|
| 투명성 (Transparency) | 구조화된 출력(Instructor) + 비용 모니터링(Helicone) |
| 책임성 (Accountability) | Human-in-the-loop(LangGraph interrupt) + Audit trail |
| 프라이버시 (Privacy) | 가드레일 PII 마스킹(NeMo Guardrails) + MCP 권한 통제 |
| 안전성 (Safety) | Red Teaming(Garak, PyRIT) + Prompt Injection 방어(LLM Guard) |
| 공정성 (Fairness) | Alignment 평가(DeepEval) + 편향 벤치마크 |
| 포용성 (Inclusion) | 토큰 절약(LiteLLM, Prompt Caching) → 비용 장벽 낮춤 → 접근성 확대 |

거버넌스가 "AI는 이래야 한다"를 말했다면, 후반기는 "이렇게 만들 수 있다"를 보여준다.

## 흥미로운 주제인 이유

1. **손으로 만져볼 수 있다**: 매주 새로운 오픈소스 도구를 다룬다.
2. **실무에 바로 쓸 수 있다**: 졸업 후 AI 서비스를 만들 때 이 도구들을 실제로 사용하게 된다.
3. **정답이 없는 트레이드오프를 다룬다**: "안전하면 느려진다", "통제하면 자유도가 줄어든다" — 엔지니어링의 본질적 긴장을 경험한다.
